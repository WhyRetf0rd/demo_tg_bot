# Telegram бот для мастера маникюра

## Что умеет бот

- Запись клиентов через inline-кнопки и FSM.
- Выбор даты/времени из доступных слотов на 1 месяц вперед.
- Ограничение: у пользователя только одна активная запись.
- Отмена записи пользователем (слот снова становится доступным).
- Админ-панель (`/admin`) для управления днями, слотами и записями.
- Автонапоминания за 24 часа до визита через APScheduler.
- Уведомления о новых/отмененных записях админу и в канал.
- Кнопка **«Главное меню»** на шагах записи (inline + reply при вводе имени/телефона); команда `/start` сбрасывает сценарий.
- В `/admin`: просмотр **активных записей на сегодня** и кнопка **повторной публикации** расписания дня в канал (`CHANNEL_ID`).

## Установка

1. Создайте и активируйте виртуальное окружение:

```bash
python -m venv venv
venv\Scripts\activate
```

2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Задайте переменные в **`.env`** (рекомендуется):

```bash
copy .env.example .env
```

Если файла `.env` нет, но вы уже заполнили **`.env.example`**, бот всё равно подхватит оттуда значения (для Git лучше хранить секреты только в `.env`, а в `.env.example` оставить заглушки).

4. Заполните переменные в `.env` (или в `.env.example`, см. выше):

- `BOT_TOKEN` — токен бота.
- `ADMIN_ID` — Telegram ID администратора.
- `CHANNEL_ID` — ID канала для уведомлений о записях (бот должен иметь право писать в канал).
- `DATABASE_PATH` — путь к SQLite базе (по умолчанию `appointments.db`).

5. Запустите бота:

```bash
python bot.py
```

## Важно

- Для автонапоминаний задачи восстанавливаются из базы при старте бота.

### Сообщения в канал (`CHANNEL_ID`)

1. Добавьте бота в канал **администратором** с правом **«Публиковать сообщения»** (Post messages).
2. Для супергрупп/каналов id обычно вида **`-100xxxxxxxxxx`** (узнать: @userinfobot, пересланное сообщение из канала и т.п.).
3. В логах при старте бота строка `Канал для уведомлений:` подтверждает доступ; если `Ошибка:` — проверьте id и права.
4. В `.env` в id **не должно быть пробелов** (скрипт их убирает).
